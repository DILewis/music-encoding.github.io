Step two: adding some attributes to the structure skeleton that you created in the previous step. These attributes will provide basic information about the staff, clef and key of the example.

The key and time signature are common for the whole score in this example, which is why you describe them in <scoreDef> regardless of how many staffs there are. You need to use the following attributes of <scoreDef>:

  • @key.sig (key signature) – the number of sharps/flats in written key signature, e.g. “1s” for 1 sharp, “2f” for 2 flats, etc.
  • @meter.sym (meter symbol) – “common” for common time (C) or “cut” for cut time (C/).

Clef is described in the <staffDef> element, because it is related to an individual staff. You need to use the following attributes of <staffDef>:

  • @n (ordinal number) – ”1” for the first staff
  • @lines – “5” for five-line staff
  • @clef.shape – the type of clef: “F”, “C” or “G”
  • @clef.line – the staff line on which the clef is placed (counting from the bottom), e.g.: “1”, “2”, “5”."
