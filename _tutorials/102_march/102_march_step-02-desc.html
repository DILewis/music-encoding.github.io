In this step, you will add some attributes to the structure skeleton that you created in the previous step. These attributes will provide basic information about the staff, clef and key of the example.

The key and time signature are common for the whole score, that’s why you describe them in <scoreDef> regardless of how many staffs there are. You need to use the following attributes of <scoreDef>:

  • key signature @key.sig – the number of sharps/flats in written key signature, e.g. “1s” for 1 sharp, “2f” for 2 flats, etc.
  • meter symbol @meter.sym – “common” for common time (C) or “cut” for cut time (C/). In case of numeric time signature you’d use other attributes: @meter.count for the top number of the meter signature and @meter.unit for the bottom number of the meter signature.

Clef is described in the <staffDef> element, because it is related to an individual staff. You need to use the following attributes of <staffDef>:

  • ordinal number @n – ”1” for the first staff
  • @lines – “5” for five-line staff
  • @clef.shape – the type of clef: “F”, “C” or “G”
  • @clef.line – the staff line on which the clef is placed (counting from the bottom), e.g.: “1”, “2”, “5”."

Add the attributes described above and their respective values to <scoreDef> and <staffDef> in the editor below.
