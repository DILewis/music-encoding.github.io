Cuando la música se presenta con varios instrumentos/voces que comparten pentagrama, a veces es necesario "empujar" a algunos elementos como las notas a una posición posterior en el compás, sin significar que se están haciendo silencios. Esto suele ocurrir cuando el documento que se pretende codificar cambia de la notación basada en acordes a las plicas separadas dentro de un compás (consultar la figura de abajo). A medida que MEI utiliza el elemento <layer> para ordenar varios instrumentos (y en estas situaciones un instrumento cambia entre esas capas a mitad de compás), los "huecos" en esos compases necesitan rellenarse. Para esto es para lo que sirve el elemento <space>.

Obviamente, un espacio no es en realidad un silencio. SIn embargo, los silencios (invisibles) se (ab)usan para tales fines en otros formatos de codificación o editores de partituras. El elemento <space> de MEI tiene el beneficio de que no tiene ningún significado semántico, simplemente va empujando ortos elementos. Esto hace que el hecho de realmente procesar una codificación y extraer voces individualmente para ello sea mucho más sencillo, ya que no se introduce falsamente ningún elemento portador de información inexistente solo para una mejor coordinación.

Sin embargo, se puede aplicar cierta duración al elemento <space> al utilizar el atributo @dur. Incluso se puede utilizar @dots para extender esa duración a la mitad de su valor. La duración que se aplica con <space> es el número de pulsaciones que algunos elementos, como una <note>, seguidos de <space> se desplazan hacia la derecha.

En el editor de abajo, se ve la codificación pre-rellenada de una nota Sol4 (posicionada en el primer tiempo de la segunda capa). Por favor, empuja esta nota hasta el final del compás, de manera que se sitúe en el último tiempo bajo el Re5, como en la imagen de arriba. Para esto, utiliza el elemento <space> y aplica la duración adecuada (en total deben ser 3 tiempos). De hecho hay varias maneras de codificarlo:

  • con tres <space> que utilicen el mismo valor para @dur,
  • con dos elementos <space> que utilicen solo @dur, pero con dos valores diferentes,
  • o incluso con un solo elemento <space>, que utilice @dur y @dots.
