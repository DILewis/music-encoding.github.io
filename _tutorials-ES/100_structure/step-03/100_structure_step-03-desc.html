Tercer paso: conocer los principales elementos hijo del elemento raíz <mei>: <meiHead> y <music>.

¡Buen trabajo! Ahora que hemos declarado un espacio de nombres dentro de nuestro elemento raíz, todos los demás elementos anidados en el elemento raíz pueden resolverse de acuerdo con el espacio de nombres MEI mediante el procesamiento automático.

Como ya se ha mencionado, se puede imaginar que un documento XML es como un árbol que se ramifica desde un elemento raíz (externo) hasta los elementos hijo anidados (internos). Un archivo codificado con MEI contiene al menos dos estructuras dentro del elemento raíz <mei>:

  • <meiHead>: contiene un conjunto de elementos que proporcionan metadatos relacionados con la codificación MEI, incluyendo información bibliográfica, declaraciones de responsabilidad, normas de codificación o procedencia.
  • <music>: contiene un conjunto de elementos que proporcionan la codificación del contenido musical real. Entre otras cosas, puede comprender elementos estructurales para describir la forma de una pieza musical (secciones, movimientos, grupos de documentos), o elementos de notación, como, por ejemplo, notas, silencios, articulaciones o dinámicas.
